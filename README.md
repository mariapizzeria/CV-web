# Сайт-резюме

## Задача

Разработать сайт-резюме для поиска работы, состоящий из двух сервисов: frontend и backend.

## Сервис backend

Реализовать следующие сервисы:
- Образование (education)
- Социальные сети (social)
- Навыки (skills)
- Опыт работы (experience)

### Сервис "Образование"

**Описание:** Добавление информации о пройденных курсах. Данные выводятся на странице edu.html.

**Эндпоинт:** `/education`

**Ответ API:**
- `course` — название законченного курса (string)
- `duration` — продолжительность обучения (string)
- `college` — учебное заведение (string)

**Пример:**
```json
{
    "duration": "2022-2023",
    "college": "Яндекс Практикум",
    "course": "Аналитика данных"
}
```


### Сервис "Социальные сети"

**Описание:** Добавление информации о социальных сетях. Данные выводятся на странице main.html.

**Эндпоинт:** `/social`

**Ответ API:** 
- `image` - путь к изображению (расположены во frontend/public/pages/image) (string) 
- `link` - ссылка на ресурс (string)

**Пример:**
```json
{
    "image":"image/logo.svg",
    "link":"https://www.google.ru"
}
```

### Сервис Навыки:

**Описание:** Добавление информации о приобретенных навыках. Данные выводятся на странице stack.html.

**Эндпоинт:** `/skills`

**Ответ API:** 
- `type` - название категории навыков (string)
- `skill` - список навыков ([]string)

**Пример:**
```json
{
    "type":"frontend",
    "skill":["Tailwind", "HTML", "CSS", "TypeScript"]
}
```

### Сервис Опыт:

**Описание:** Добавление информации об опыте работы. Данные выводятся на странице exp.html.

**Эндпоинт:** `/experience`

**Ответ API:** 
- `title` - место работы и должность (string)
- `duration` - период работы (string)
- `description` - описание обязанностей (string)

**Пример:**
```json
{
    "title":"Фриланс: fullstack - разработчик",
    "duration":"октябрь 2025 - апрель 2026",
    "description":"Длинное описание с использованием тегов для верстки"
}
```

### Возможности API:

В сервисах должны быть реализованы методы: GET, POST, PUT и DELETE.

Методы POST, PUT и DELETE требуют авторизации через API Key.

### База данных

- PostgreSQL

**Таблицы**: education, social, experience, skills
